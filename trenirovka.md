git pull origin main --rebase

Делает три вещи по очереди:

git fetch — скачивает коммиты с GitHub на твой компьютер, но ничего не меняет
git rebase — берёт твои локальные коммиты и «перекладывает» их поверх скачанных. История становится линейной: сначала чужое, потом твоё
Результат готов к пушу
Без --rebase была бы команда git pull — она делает то же самое, но вместо rebase создаёт «merge commit» — дополнительный коммит-слияние. История выглядит как разветвление и соединение. Rebase чище.

git push -u origin main

git push — отправить локальные коммиты на GitHub
origin — псевдоним для URL репозитория (https://github.com/zeromode15rus/english_learning.git)
main — ветка которую отправляем
-u — запомнить связь между локальной main и удалённой origin/main. После этого достаточно просто git push — без лишних аргументов

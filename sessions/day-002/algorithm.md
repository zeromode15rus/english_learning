## Алгоритм — Мемоизация (Memoization)

### Интуиция

Представь, что тебя попросили вычислить 15-е число Фибоначчи. Наивный рекурсивный алгоритм вычислит fib(13) дважды, fib(12) — четырежды, fib(2) — сотни раз. Это катастрофа: экспоненциальное дублирование работы. Мемоизация — это простая идея: первый раз вычисли и запомни результат; второй раз просто достань из кэша.

Мемоизация — частный случай динамического программирования «сверху вниз» (top-down): рекурсия сохраняет, а не воспроизводит работу.

### Трассировка: fib(5)

Без мемоизации дерево вызовов:
```
fib(5)
├── fib(4)
│   ├── fib(3)
│   │   ├── fib(2) → 1
│   │   └── fib(1) → 1
│   └── fib(2) → 1   ← повтор
└── fib(3)           ← повтор
    ├── fib(2) → 1   ← повтор
    └── fib(1) → 1
```
Итого вызовов: 15

С мемоизацией — каждое значение вычисляется ровно один раз:
```
fib(5) → вычисляем → cache[5] = 5
fib(4) → вычисляем → cache[4] = 3
fib(3) → вычисляем → cache[3] = 2
fib(2) → вычисляем → cache[2] = 1
fib(1) → базовый → 1
fib(2) → cache hit → 1
fib(3) → cache hit → 2
```
Итого вызовов: 9 (для n=5); в общем случае O(n).

### Реализация на JavaScript

```js
function memoize(fn) {
  const cache = new Map();
  return function (...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

const fib = memoize(function (n) {
  if (n <= 1) return n;
  return fib(n - 1) + fib(n - 2);
});

console.log(fib(50)); // 12586269025 — мгновенно
```

Обрати внимание: `fib` ссылается на саму себя по имени — это обязательно для мемоизированной рекурсии. Если передать анонимную функцию в `memoize`, рекурсивные вызовы внутри пойдут в немемоизированную версию.

### Big O

| | Время | Память |
|---|---|---|
| Без мемоизации | O(2ⁿ) | O(n) — стек |
| С мемоизацией | O(n) | O(n) — кэш |

### Применение в React и Redux

**React.memo** — мемоизация компонента целиком: повторный рендер не происходит, если props не изменились (shallow comparison).
```js
const Button = React.memo(({ label, onClick }) => <button onClick={onClick}>{label}</button>);
```

**useMemo** — мемоизация вычисленного значения. Пересчёт только когда изменились зависимости.
```js
const sortedList = useMemo(() => [...items].sort(compareFn), [items]);
```

**useCallback** — мемоизация функции. Нужна, чтобы ссылка на функцию не менялась при каждом рендере (важно при передаче в дочерний компонент с React.memo).
```js
const handleClick = useCallback(() => dispatch(action), [dispatch]);
```

**Reselect (Redux)** — мемоизированные селекторы. Вычисляет производное состояние ровно тогда, когда изменились входные данные.
```js
const selectFilteredTodos = createSelector(
  [selectTodos, selectFilter],
  (todos, filter) => todos.filter(t => t.status === filter)
);
```

### Когда применять — когда нет

**Применять:**
- Рекурсивные функции с перекрывающимися подзадачами (Fibonacci, coin change, longest common subsequence)
- Дорогостоящие вычисления в React-компонентах, вызываемых часто (длинные сортировки, фильтры больших списков)
- Redux-селекторы, которые получают весь стор и вычисляют производное состояние

**Не применять:**
- Если функция вызывается каждый раз с разными аргументами — кэш не даёт хита, но занимает память
- Если вычисление дешёвое (простое сложение) — накладные расходы на `Map` и `JSON.stringify` превышают выигрыш
- Не заменять мемоизацией реальную оптимизацию алгоритма: O(n²) с кэшем — это не то же самое, что O(n)
